# SVG 工作流程图基础标准

本标准用于判断一个 SVG 工作流程图是否基本适合进入技术课程或开发者文档。

这是基础标准，只包含最容易想到的要求。它并不完美，更细致的专业要求见补充标准。

## 一、基础要求

1. 输出必须是合法 SVG。
2. SVG 必须包含 `viewBox`。
3. 画布比例建议为 16:9。
4. 不允许嵌入 `<image>`、base64 或外部位图。
5. SVG 能直接用浏览器打开。
6. SVG 中的文字、形状、线条应可编辑。

## 二、语义要求

一张合格的工作流程图必须让读者看懂：

1. 流程从哪里开始；
2. 每一步做什么；
3. 哪些节点是核心动作；
4. 哪些节点是工具、系统或外部资源；
5. 哪里产生反馈；
6. 什么时候结束；
7. 如果失败，下一步回到哪里。

## 三、布局要求

1. 主流程方向清楚，可以从左到右、从上到下，或环形闭环。
2. 主流程节点建议 5-8 个。
3. 节点之间留出足够间距。
4. 连线不能穿过主要文字。
5. 文字不能和图形明显重叠。
6. 主体区域应覆盖画布主要区域，不能太小或挤在角落。
7. 画布四周不要留下大面积空白或装饰性边框。外边距应克制，主体应尽量贴近可用画布范围。
8. 线条应服务流程关系，不画无意义装饰线。
9. 箭头应指向明确，不能让读者猜流程方向。

## 四、文字要求

1. 文字用于标记节点，不用于解释全部课程内容。
2. 节点标题建议 2-6 个汉字。
3. 单条中文文本不超过 14 个汉字。
4. 文字总组数建议不超过 14 组。
5. 不写长段说明。
6. 不把课文讲解塞进图里。
7. 文字必须清晰可读，字号不能过小。主要节点文字应明显可读，不要为了塞内容把字缩得很小。
8. 同一层级文字应使用一致字号和字重。

## 五、风格要求

1. 整体风格应简洁、大方、技术感强。
2. 采用浅色科技风：白色或极浅背景，低饱和色块，细线条，轻阴影。
3. 视觉应有现代产品界面的克制感：留白充足、层级清楚、元素对齐、圆角统一。
4. 背景使用白色或透明，不使用深色大背景。
5. 主色使用蓝色，推荐 `#0033CC`。
6. 可使用蓝色渐变，例如 `#0033CC -> #3B82F6`，但渐变应克制。
7. 强调色使用橙色，用于风险、反馈、关键节点或边界提示。
8. 可使用灰色和黑色构成文字层级。
9. 不使用复杂人物、装饰性渐变球、噪声背景。
10. 颜色数量应克制，建议不超过 8 种。

## 六、动态与交互要求

如果任务要求动态图，应满足：

1. 主流程箭头或反馈闭环应有轻量动画。
2. 动画用于表达流程推进，不做纯装饰。
3. 至少一个节点支持 hover、click 或 data-step 演化逻辑。
4. 动画和交互必须能在浏览器中预览。
5. 动效应克制，不能影响阅读。
